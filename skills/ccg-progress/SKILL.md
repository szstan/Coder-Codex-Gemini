# CCG 项目进度管理

## 概述

CCG 项目进度管理系统提供完整的项目进度跟踪、任务管理、风险识别和报告生成能力。

### 核心功能

1. **进度跟踪**：自动跟踪版本、里程碑、任务完成情况
2. **任务管理**：管理活跃任务、已完成任务、优先级排序
3. **风险管理**：识别和跟踪风险、阻塞项
4. **报告生成**：自动生成进度报告、统计数据
5. **可视化**：提供清晰的进度可视化展示

---

## 文件结构

```
PROJECT_ROADMAP.md          # 项目路线图（版本规划、里程碑）
.ccg/progress.json          # 进度跟踪配置（实时数据）
skills/ccg-progress/        # 进度管理 Skill
```

## 使用场景

### 场景 1：查看项目总体进度

**触发时机**：
- 会话启动时（可选）
- 用户询问项目进度
- 完成重要里程碑后

**执行步骤**：
```
1. 读取 .ccg/progress.json
2. 显示当前版本进度
3. 显示里程碑完成情况
4. 显示活跃任务列表
5. 显示风险和阻塞项
```

**输出格式**：
```
📊 **项目进度概览**

**当前版本**: v1.1.0 (60% 完成)
**目标发布**: 2026-02-01 (剩余 12 天)

**里程碑进度**:
- ✅ 会话管理系统 (100%)
- 🚧 进度管理系统 (30%)
- ✅ Codex Review 模式 (100%)
- ✅ 文档优化 (100%)

**活跃任务** (5 个):
- 🔴 创建进度跟踪配置 (进行中)
- 🔴 创建进度管理 Skill (待开始)
- 🟡 集成进度管理到全局配置 (待开始)

**风险** (2 个):
- 🟡 进度数据结构设计复杂
- 🟡 自动进度更新逻辑复杂

**阻塞项**: 无
```

## 场景 2：更新任务状态

**触发时机**：
- 任务开始时
- 任务完成时
- 任务状态变化时

**执行步骤**：
```
1. 读取 .ccg/progress.json
2. 找到对应任务
3. 更新任务状态
4. 更新里程碑进度
5. 重新计算完成百分比
6. 保存更新后的配置
```

**示例**：
```
任务 "创建进度跟踪配置" 已完成
- 状态: pending → completed
- 完成时间: 2026-01-20
- 里程碑 "进度管理系统" 进度: 20% → 40%
- 版本 v1.1.0 进度: 60% → 68%
```

## 自动化集成

### 与 CCG 工作流集成

**会话启动时**（可选）：
```
1. 加载项目上下文 (.ccg/project-context.json)
2. 加载进度数据 (.ccg/progress.json)
3. 显示简要进度概览
4. 提示是否需要查看详细进度
```

**任务完成时**：
```
1. Claude 验收通过后
2. 自动更新任务状态为 completed
3. 更新里程碑进度
4. 重新计算版本完成百分比
```

**里程碑完成时**：
```
1. 所有任务完成后
2. 标记里程碑为 completed
3. 生成里程碑完成报告
4. 提示用户进入下一里程碑
```

## 进度数据维护

### 手动更新

**更新任务状态**：
```json
// 在 .ccg/progress.json 中找到任务
{
  "id": "task-001",
  "status": "pending"  // 改为 "in_progress" 或 "completed"
}
```

**添加新任务**：
```json
{
  "id": "task-new",
  "title": "新任务标题",
  "priority": "high",
  "status": "pending",
  "assignee": "Claude",
  "created_at": "2026-01-20",
  "due_date": "2026-01-21",
  "milestone": "m1.1.2",
  "tags": ["tag1", "tag2"]
}
```

**添加风险**：
```json
{
  "id": "risk-new",
  "title": "风险描述",
  "severity": "high",
  "impact": "影响说明",
  "mitigation": "缓解措施",
  "status": "monitoring",
  "created_at": "2026-01-20"
}
```

## 最佳实践

### 1. 定期更新进度

**建议频率**：
- 每日：更新任务状态
- 每周：生成进度报告
- 里程碑完成：更新路线图

### 2. 及时识别风险

**关键指标**：
- 任务延期超过 2 天 → 标记为风险
- 阻塞项超过 1 天 → 升级为高优先级
- 完成率低于预期 → 调整计划

### 3. 保持数据准确

**数据质量**：
- 任务状态实时更新
- 完成时间准确记录
- 风险描述清晰具体

## 常见问题

**Q: 如何查看项目总体进度？**
A: 执行 `/ccg-progress` Skill，或直接查看 `.ccg/progress.json`

**Q: 如何添加新任务？**
A: 手动编辑 `.ccg/progress.json`，添加到 `active_tasks` 数组

**Q: 如何标记任务完成？**
A: 将任务状态改为 `completed`，并移动到 `completed_tasks` 数组

**Q: 进度数据会自动更新吗？**
A: 部分自动（任务完成时），部分手动（添加新任务、风险）
