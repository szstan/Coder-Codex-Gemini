# CCG 项目接手 Skill

> **用途**：自动化执行项目接手流程，快速理解项目并建立开发环境

## 触发条件

当检测到以下情况时，自动触发此 Skill：

1. 用户说"接手这个项目"、"分析这个项目"
2. 用户说"我是新加入的"、"第一次接触这个项目"
3. 用户要求"了解项目状态"、"评估项目进度"

---

## 执行流程

### 阶段 1：项目扫描（5-10 分钟）

**Claude 自动执行**：
1. 读取项目根目录文件列表
2. 识别项目类型（Python/Node.js/Java/Go）
3. 读取 README.md 和主要文档
4. 分析 Git 历史（最近 20 次提交）
5. 记录到 .ccg/project-context.json

**向用户报告**：
```
📊 项目扫描完成

**项目类型**：Python Web 应用
**技术栈**：FastAPI + PostgreSQL
**最近活动**：3 天前（修复支付 Bug）
**主要贡献者**：Alice, Bob

正在进入环境检查阶段...
```

### 阶段 2：环境检查（10-15 分钟）

**Claude 自动执行**：
1. 运行 `.ccg/verify-environment.sh`
2. 检查系统依赖
3. 尝试安装项目依赖
4. 验证环境完整性

**向用户报告**：
```
🔧 环境检查完成

✅ 通过：8 项
⚠️  警告：2 项（Docker 未安装）
❌ 失败：0 项

**建议**：安装 Docker 以支持容器化部署

正在进入进度评估阶段...
```

### 阶段 3：进度评估（10-15 分钟）

**Claude 自动执行**：
1. 统计代码指标
2. 检查测试覆盖率
3. 查找 TODO/FIXME
4. 识别未完成功能
5. 生成项目状态快照

**向用户报告**：
```
📈 进度评估完成

**整体完成度**：60%
**测试覆盖率**：75%
**待处理 TODO**：23 个
**待修复 FIXME**：8 个

**未完成功能**：
- 支付集成（40%）
- 通知系统（30%）
- 管理后台（0%）

正在生成接手报告...
```

### 阶段 4：生成接手报告（5 分钟）

**Claude 自动执行**：
1. 汇总所有信息
2. 生成完整的接手报告
3. 提供下一步建议

**向用户展示完整报告**（参考 ai/project_handover_guide.md 中的模板）
