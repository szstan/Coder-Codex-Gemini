# CCG 项目接手检查清单

> **用途**：系统化的项目接手检查清单，确保不遗漏关键步骤

## 使用说明

- ✅ 已完成
- 🔄 进行中
- ⏸️ 暂停/跳过
- ❌ 未开始

---

## 第一阶段：项目理解（预计 30-60 分钟）

### 1.1 项目基本信息

- [ ] 读取 README.md
- [ ] 识别项目类型（Python/Node.js/Java/Go）
- [ ] 记录项目名称和版本
- [ ] 查看项目许可证

### 1.2 技术栈识别

- [ ] 识别编程语言和版本
- [ ] 识别主要框架
- [ ] 识别数据库类型
- [ ] 识别缓存/消息队列
- [ ] 记录到 `.ccg/project-context.json`

### 1.3 项目结构分析

- [ ] 查看目录结构
- [ ] 识别源代码目录
- [ ] 识别测试目录
- [ ] 识别配置目录
- [ ] 识别文档目录

### 1.4 Git 历史分析

- [ ] 查看最近 20 次提交
- [ ] 识别主要贡献者
- [ ] 查看活跃分支
- [ ] 识别最近修改的核心文件

---

## 第二阶段：环境建立（预计 30-90 分钟）

### 2.1 系统依赖检查

- [ ] 检查 Git 是否安装
- [ ] 检查编程语言运行时（Python/Node.js/Java/Go）
- [ ] 检查包管理器（pip/npm/maven/cargo）
- [ ] 检查 Docker（如果需要）
- [ ] 运行 `.ccg/verify-environment.sh`

### 2.2 项目依赖安装

- [ ] 安装核心依赖（requirements.txt / package.json）
- [ ] 安装开发依赖（requirements-dev.txt / devDependencies）
- [ ] 验证依赖完整性（pip check / npm list）
- [ ] 记录安装的包列表到 `.ccg/installed-packages.txt`

### 2.3 配置文件管理

- [ ] 查找配置文件（.env / config.yaml）
- [ ] 检查是否有配置模板（.env.example）
- [ ] 创建本地配置文件
- [ ] 记录必需配置项到 `.ccg/config-checklist.md`

### 2.4 数据库和服务

- [ ] 检查数据库连接配置
- [ ] 运行数据库迁移（如果有）
- [ ] 检查 Redis/缓存服务（如果有）
- [ ] 检查消息队列（如果有）

### 2.5 环境验证

- [ ] 运行环境验证脚本
- [ ] 尝试启动应用
- [ ] 运行测试套件
- [ ] 记录环境状态

---

## 第三阶段：进度把控（持续）

### 3.1 代码状态评估

- [ ] 统计代码文件数和行数
- [ ] 查找 TODO/FIXME 数量
- [ ] 检查测试覆盖率
- [ ] 识别未完成功能

### 3.2 Git 和分支管理

- [ ] 查看活跃分支
- [ ] 查看未合并的 PR
- [ ] 查看 Open Issues
- [ ] 识别功能分支状态

### 3.3 项目状态快照

- [ ] 生成项目状态快照（.ccg/project-snapshot.json）
- [ ] 记录当前完成度
- [ ] 记录下一个里程碑
- [ ] 识别风险和阻塞

### 3.4 生成接手报告

- [ ] 生成完整的接手报告
- [ ] 向用户展示报告
- [ ] 确认下一步行动计划

---

## 快速检查命令

```bash
# 一键运行所有检查
bash .ccg/verify-environment.sh

# 查看项目状态
cat .ccg/project-snapshot.json

# 查看配置清单
cat .ccg/config-checklist.md
```
